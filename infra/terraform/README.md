# Инструкция по развертыванию инфраструктуры с помощью Terraform

В этой папке находятся конфигурационные файлы для автоматического создания всей необходимой инфраструктуры Google Cloud Platform (GCP) для проекта «Тендерный помощник ИП».

---

## 🛠️ Подготовка

### 1. Установка Terraform
Если Terraform еще не установлен на вашем компьютере:
- Скачайте его с [официального сайта Terraform](https://developer.hashicorp.com/terraform/downloads).
- Распакуйте исполняемый файл `terraform.exe` и добавьте его в системную переменную окружения `PATH` (либо поместите в любую папку и запускайте с указанием полного пути).
- Проверьте установку в консоли:
  ```bash
  terraform -version
  ```

### 2. Получение ключа доступа к GCP (gcp-creds.json)
Для того чтобы Terraform мог создавать ресурсы от вашего имени:
1. Перейдите в Google Cloud Console.
2. Откройте раздел **IAM & Admin → Service Accounts** (Сервисные аккаунты).
3. Найдите существующий аккаунт с административными правами (или создайте новый, например, `terraform-admin` с ролью `Owner` или `Editor` + `Project IAM Admin`).
4. Нажмите на три точки напротив аккаунта и выберите **Manage keys → Add key → Create new key**.
5. Выберите тип ключа **JSON** и нажмите **Create**. Файл автоматически скачается на ваш компьютер.
6. Переименуйте скачанный файл в `gcp-creds.json` и сохраните его прямо в эту папку: `infra/terraform/gcp-creds.json`.
> [!WARNING]
> Файл `gcp-creds.json` содержит приватные ключи вашего облака. Он **уже добавлен** в файл `.gitignore`, поэтому никогда не попадет в публичный репозиторий GitHub.

### 3. Включение API в Google Cloud
Убедитесь, что в вашем проекте GCP включены следующие API:
- **Compute Engine API** (`compute.googleapis.com`)
- **Cloud SQL Admin API** (`sqladmin.googleapis.com`)
- **Service Networking API** (`servicenetworking.googleapis.com`) — требуется для безопасного соединения с базой данных внутри внутренней сети VPC.

## 🚀 Развертывание ресурсов

Откройте терминал (PowerShell или командную строку), перейдите в эту директорию и выполните по шагам:

### Шаг 1: Инициализация Terraform
Скачает необходимые плагины и провайдер Google Cloud:
```bash
cd infra/terraform
terraform init
```

### Шаг 2: Проверка плана создания
Покажет список ресурсов, которые будут созданы (сеть, база данных, виртуальная машина, облачное хранилище):
```bash
terraform plan
```

### Шаг 3: Применение конфигурации
Запустит процесс создания инфраструктуры в GCP:
```bash
terraform apply
```
*На запрос подтверждения введите `yes` и нажмите Enter.*

> [!NOTE]
> Создание Cloud SQL (базы данных PostgreSQL) обычно занимает от 5 до 10 минут. Пожалуйста, дождитесь окончания работы скрипта.

## 📋 Результаты развертывания
По завершении команды в консоли отобразятся результаты (`outputs`):
- `vm_public_ip` — публичный IP-адрес вашей VM, к которой можно подключаться по SSH.
- `db_private_ip` — внутренний IP-адрес базы данных PostgreSQL (используется бэкендом).
- `gcs_bucket_name` — имя созданного облачного хранилища документов.
